Part - A (2 Marks Questions)

1.What is software Engineering
Software Engineering is the disciplined and systematic approach to the development, operation, and maintenance of software, using engineering principles to build high-quality, cost-effective, and maintainable software systems. .
2.What is Software Development Lifecycle (or) Software Process? List out its phases.
Software Development Lifecycle (SDLC) is a systematic process for planning, creating, testing, deploying, and maintaining software. It defines the phases involved in the development of a software product from initial idea to final deployment and maintenance. Phases of SDLC: 1. Requirement Analysis 2. System Design 3. Implementation (Coding) 4. Testing 5. Deployment 6. Maintenance .
3. What is XP (Extreme Programming) ?
Extreme Programming (XP) is an agile software development methodology that emphasizes frequent releases, customer collaboration, continuous testing, and simple design. It focuses on improving software quality and flexibility to changing requirements.
4.Recall agile process?
The Agile Process is a flexible and iterative software development approach that emphasizes collaboration, customer feedback, and rapid delivery of small, working software increments.
5.List out the Software design patterns.
1. Creational Patterns – Singleton, Factory Method, Abstract Factory, Builder, Prototype 2. Structural Patterns – Adapter, Decorator, Composite, Proxy, Facade, Bridge, Flyweight 3. Behavioral Patterns – Observer, Strategy, Command, Iterator, State, Template Method, Visitor, Mediator, Memento
6.What is Software requirement specification?
Software Requirement Specification (SRS) is a formal document that describes the functional and non-functional requirements of a software system. It defines what the software should do, how it should perform, and serves as a contract between the client and the development team.
7.List out the Various diagrams used in object modeling using UML
The various diagrams used in object modeling using UML are: 1. Class Diagram 2. Object Diagram 3. Use Case Diagram 4. Sequence Diagram 5. State Diagram 6. Activity Diagram
8.What are the types of requirements?
The main types of requirements are: 1. Functional Requirements – Specify what the system should do (features and functions). 2. Non-Functional Requirements – Define how the system performs (performance, security, usability).
9. Define:finite state machine
A Finite State Machine (FSM) is a computational model consisting of a finite number of states, transitions between those states, and actions. It starts in an initial state and changes states based on input, following defined rules.
10.List out the software design patterns.
The main types of software design patterns are: 1. Creational Patterns – e.g., Singleton, Factory, Abstract Factory 2. Structural Patterns – e.g., Adapter, Decorator, Composite 3. Behavioral Patterns – e.g., Observer, Strategy, Command
11. What is mean by adapter in software design?
In software design, an Adapter is a structural design pattern that allows incompatible interfaces to work together. It acts as a bridge between two classes by converting the interface of one class into an interface expected by the client.
12.what is publish-subscribe in software design ?
In software design, Publish-Subscribe (Pub-Sub) is a messaging pattern where publishers send messages without knowing the recipients, and subscribers receive messages based on interest, without knowing the sender. It promotes loose coupling between components.
13.What is proxy,façade in software design?
In software design: • Proxy is a structural design pattern that provides an object representing another object, controlling access to it. It can be used for lazy initialization, access control, or logging. • Facade is a structural design pattern that provides a simplified interface to a complex subsystem, making it easier to use by hiding the subsystem's complexity.
14..What is UI design
UI (User Interface) Design is the process of designing the layout, visual elements, and interactive components of a software application or website. It focuses on creating an intuitive and aesthetically pleasing experience for users, ensuring ease of navigation and effective interaction.
15.What is symbolic execution?
Symbolic execution is a program analysis technique where a program is executed with symbolic inputs (instead of actual data). The execution explores multiple paths by using symbols to represent values, allowing the analysis of different possible outcomes and detecting potential errors or vulnerabilities without running the program with specific inputs.
16. What is model checking ?
Model checking is an automated verification technique used in software engineering and hardware design to check whether a system (typically represented as a model) satisfies certain specifications or properties. It systematically explores all possible states of a system to ensure it adheres to predefined rules or behaviors, often focusing on aspects like safety, liveness, and correctness.
17. Define :Debugging
Debugging is the process of identifying, isolating, and fixing bugs or errors in software. It involves analyzing the code, running tests, and using tools to track down and resolve issues that cause the program to behave incorrectly or unexpectedly.
18.what is regression testing
Regression testing is the process of testing software after changes (such as enhancements, bug fixes, or updates) to ensure that new code does not negatively impact the existing functionality. It helps verify that previously working features continue to function as expected.
19.Write the difference between black box and white box testing?
Black Box Testing: • Focuses on testing the functionality of the software without knowledge of its internal code or structure. • Testers validate inputs and outputs, ensuring the system meets the specified requirements. White Box Testing: • Involves testing the internal structure, logic, and code of the software. • Testers need knowledge of the source code to create test cases, checking conditions like code paths and loops.
20.What is Software project management?
Software Project Management is the process of planning, organizing, and overseeing the development and delivery of software projects. It involves managing resources, timelines, budgets, and risks, ensuring the project meets its objectives, is completed on time, and stays within scope.
21.. Define :Software configuration management
Software Configuration Management (SCM) is the process of systematically managing and controlling changes to software and its associated artifacts throughout the software development lifecycle. It ensures the integrity, traceability, and consistency of software versions, configurations, and documentation.
22.What is project scheduling?
Project scheduling is the process of planning and organizing tasks, resources, and timelines for a project. It involves defining project milestones, setting deadlines, and determining the sequence of activities to ensure the project is completed on time and within scope. It helps track progress and manage dependencies between tasks.
23..Add a note on :cloud as a platform
loud as a Platform refers to using cloud computing services to develop, deploy, and manage applications over the internet. It offers scalable resources, flexibility, and cost efficiency without the need for physical infrastructure. Popular examples include AWS, Microsoft Azure, and Google Cloud, which provide services like computing, storage, and networking to support business needs.
24.. what are deployment tools?
Deployment tools are software applications or platforms that automate the process of moving code from development to production environments. They help streamline the deployment process, ensuring consistency, speed, and error-free releases. Examples include Jenkins, Docker, and Kubernetes, which manage builds, testing, and deployment pipelines efficiently.
